# AJN Windows Desktop

تطبيق Electron يفتح لوحة إدارة AJN الحالية ويستخدم قاعدة Supabase نفسها. لا يحتوي التطبيق على Service Role أو أي مفتاح سري.

## التشغيل

```bash
pnpm desktop:dev
```

يمكن توجيه نسخة التطوير أو البناء إلى رابط مختلف عند الحاجة:

```bash
AJN_DESKTOP_URL=https://example.com/admin/dashboard pnpm desktop:dev
```

الرابط الافتراضي في النسخة المنشورة هو `https://alijan-koshat.vercel.app/admin/dashboard`.

## البناء

```bash
pnpm run typecheck
pnpm run build
pnpm desktop:build
pnpm desktop:dist
```

النواتج المتوقعة داخل `desktop/release`:

- `AJN-Setup.exe`: مثبت Windows مع اختصار سطح المكتب وقائمة Start.
- `AJN-Portable.exe`: نسخة محمولة.

يمكن بناء ملفات Windows على جهاز Windows أو CI مخصص لـ Windows. بناء `desktop:build` على macOS ينتج حزمة macOS للاختبار المحلي، بينما `desktop:dist` يستهدف Windows x64 صراحة.

## ما يعمل دون إنترنت

بعد فتح صفحات الإدارة مرة واحدة، يحتفظ Service Worker بنسخة واجهة أساسية. داخل Electron تُحفظ عمليات JSON المدعومة محلياً أولاً في SQLite ثم تزامن تلقائياً:

- طلبات المتجر والإيجار.
- حجوزات الكوشات والخدمات والتصوير.
- فواتير المبيعات والمشتريات.
- سندات القبض والصرف والحركات المالية.
- تحويلات المخزون والمهام وتغييرات الحالة الواقعة تحت مسارات هذه الوحدات.

رفع الصور والفيديو والملفات يحتاج اتصالاً بالإنترنت، لأن الملفات الكبيرة لا تدخل Queue النصية. عرض بيانات لم تُفتح سابقاً، تسجيل الدخول الأول، والإجراءات التي تتطلب تحققاً فورياً من الخادم تحتاج إنترنت أيضاً.

## الأمان والمزامنة

- جلسة الدخول محفوظة في Chromium session الخاصة بالتطبيق وليست في ملف نصي.
- محتوى Queue وإعدادات الجهاز مشفرة عبر `safeStorage`، والذي يستخدم DPAPI على Windows عندما يكون متاحاً.
- كل عملية لها `idempotency_key` وتُقبل مرة واحدة في API.
- التعارضات المالية لا تدمج تلقائياً؛ تظهر في `/admin/sync-center` ليعتمد المدير نسخة الخادم أو يعيد إرسال النسخة المحلية.
- SQLite في مجلد `userData` الخاص بالتطبيق، وتُنشأ نسخة يومية مع الاحتفاظ بآخر 7 نسخ.

## الطباعة والتحديث

الطباعة تستخدم قوالب AJN الحالية وتدعم A4 و80mm و58mm والطابعة الافتراضية والطباعة الصامتة عند تفعيلها من مركز المزامنة. مصمم REPX الحالي يعمل داخل التطبيق ويحوّل القالب إلى HTML/React دون DevExpress Runtime.

بنية `electron-updater` موجودة لكنها معطلة افتراضياً. لا يتم فحص قناة خارجية إلا عند ضبط `AJN_UPDATE_ENABLED=1` وتجهيز مزود إصدار موثوق.
