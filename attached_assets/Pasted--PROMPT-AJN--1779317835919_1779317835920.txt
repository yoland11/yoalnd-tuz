🚀 PROMPT — فصل نظام دخول الإدارة عن دخول الزبائن داخل مشروع AJN

أريد تعديل نظام الدخول الحالي داخل مشروع AJN بحيث يصبح هناك نظامين منفصلين بالكامل:

1) دخول الإدارة والموظفين
2) دخول الزبائن

بدون كسر أي ميزة موجودة.

━━━━━━━━━━━━━━
1) دخول الإدارة والموظفين
━━━━━━━━━━━━━━

أنشئ صفحة مستقلة خاصة بالإدارة فقط:

/admin/login

هذه الصفحة مخصصة فقط لـ:
- الأدمن
- الموظفين

طريقة الدخول:
- اسم مستخدم
- كلمة مرور

الشروط:
- استخدام الجلسات الحالية HttpOnly Cookie.
- استخدام bcrypt.
- حماية جميع صفحات /admin.
- إذا لم يسجل الدخول يتم تحويله تلقائياً إلى:
/admin/login

بعد تسجيل الدخول:
- يتم تحويل الأدمن إلى:
/admin/dashboard

━━━━━━━━━━━━━━
صلاحيات الموظفين
━━━━━━━━━━━━━━

إخفاء الأقسام حسب الصلاحيات:
- الطلبات
- الحجوزات
- المتجر
- الخدمات
- المنتجات
- الصور
- العملاء
- التوصيل
- الموظفين
- الإعدادات
- الفواتير
- الواتساب

إذا الموظف لا يملك صلاحية:
- لا يظهر القسم.
- وإذا دخل الرابط مباشرة يتم منعه.

━━━━━━━━━━━━━━
2) دخول الزبائن
━━━━━━━━━━━━━━

أنشئ نظام منفصل للزبائن فقط:

/account

هذا النظام خاص بالزبائن وليس الإدارة.

طريقة الدخول:
- رقم الهاتف فقط
- OTP / رمز تحقق

السلوك:
- الزبون يدخل رقم الهاتف
- يتم إرسال رمز OTP
- يدخل الرمز
- يتم تسجيل دخوله

بعد الدخول:
- يرى:
  - طلباته
  - تتبع الطلب
  - الطلبات السابقة
  - الفواتير
  - الصور
  - الملفات

━━━━━━━━━━━━━━
3) فصل كامل بين النظامين
━━━━━━━━━━━━━━

المطلوب:
- الإدارة لا تستخدم OTP.
- الزبائن لا يدخلون إلى /admin.
- الزبائن لا يرون لوحة الإدارة.
- الإدارة لا تدخل من صفحة الزبائن.

━━━━━━━━━━━━━━
4) الواجهة
━━━━━━━━━━━━━━

أضف:
- زر دخول الإدارة داخل Footer أو Header صغير.
- زر حسابي للزبائن.

مثال:
- حسابي → /account
- الإدارة → /admin/login

━━━━━━━━━━━━━━
5) الحماية
━━━━━━━━━━━━━━

- حماية جميع API الخاصة بالإدارة.
- حماية بيانات الزبائن.
- منع أي زبون من رؤية بيانات غيره.
- استخدام Rate Limit للـ OTP.
- الجلسات تكون آمنة.

━━━━━━━━━━━━━━
6) التصميم
━━━━━━━━━━━━━━

- Luxury RTL
- Black / Gold / White
- Responsive
- Minimal
- حديث وفخم

━━━━━━━━━━━━━━
7) مهم جداً
━━━━━━━━━━━━━━

- لا تكسر المتجر.
- لا تكسر التتبع.
- لا تكسر الطلبات.
- لا تكسر الواتساب.
- لا تكسر الفواتير.
- لا توجد Console Errors.
- لا توجد Build Errors.

تأكد من:
npm run typecheck
npm run build

━━━━━━━━━━━━━━
النتيجة المطلوبة
━━━━━━━━━━━━━━

أريد:
- نظام دخول إدارة مستقل.
- نظام دخول زبائن مستقل.
- فصل كامل بين الإدارة والزبائن.
- حماية كاملة وصلاحيات حقيقية.
